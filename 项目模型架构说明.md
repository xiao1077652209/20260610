> **2026-07-16 改进**：小波辅助损失权重从 0.05 提升至 0.15；早停耐心从 14 降至 10；权重衰减从 1e-4 提升至 3e-4；启用小波去噪通道（3 通道）；融合 Dropout 从 0.15 降至 0.10。

## 1. 当前模型定位

当前默认模型不是"真实图像 + 光谱"的多模态模型，而是一个双视图光谱分类模型：

- 主视图：同一条近红外光谱经预处理后的 1D 信号；
- 辅助视图：从该预处理光谱构造的小波频带信号；
- 融合方式：保留主光谱特征直通，小波特征以受控门控残差的方式补充；
- 最终默认分类器：神经网络融合特征上的 Linear SVM。

端到端训练中的 FC 分类头用于监督主光谱编码器、小波编码器和融合模块。训练结束后，代码恢复验证集最佳权重，提取特征，并用 Linear SVM 重新训练和测试。因此，默认报告的测试 Accuracy、Precision、Recall、F1 来自 Linear SVM，不是 FC 头的 argmax。

项目仍支持以下可选模式：

- `wavelet_multiview`：当前默认，小波双视图；
- `multimodal`：二维编码图像 + 光谱；
- `spectral_only`：仅主光谱；
- `image_only`：仅二维编码图像。

## 2. 当前默认数据与总体流程

默认数据集为 `dataset/20260109_286_品种_8.csv`。当前文件实际包含 287 个样本、8 个类别、1555 个原始光谱波段；**品种和产地数据集已取消强制重采样**，保留原始波段数（品种 1555 波段，产地 535 波段）。

当前 `strict` 协议（品种数据集）的实测形状如下：

```text
原始数据:                    [287, 1555]
（取消重采样，保留原始波段）
分层划分后:                  train=200, val=57, test=30
snv+d1+sg 后:                各集合最后一维由 1555 变为 1554
仅对训练集过采样后:          train=700（8 类各约 87-88 个）
小波输入:                    [B, 2, 1554]
主光谱输入:                  [B, 1, 1554]
两个分支特征/融合特征:       [B, 256]（feature_dim 从 512 降至 256）
FC embedding:                [B, 128]
FC logits:                   [B, 8]
```

总体数据流：

```text
CSV: 标签列 + 光谱列
        |
        v
读取标签并编码为 0..K-1；必要时线性重采样至目标波段数
        |
        v
原始数据先分层划分 7:2:1
        |
        v
三个集合分别进行预处理
        |
        v
仅对预处理后的训练集过采样
        |
        v
         预处理光谱 [B, 777]（当前默认）
                      |
              +-------+-------+
              |               |
              v               v
主光谱 [B,1,777]       小波视图 [B,2,777]
              |               |
              v               v
MultiScaleSpectralCNN   AttentionCNN1D
              |               |
         512 维主特征      512 维小波特征
              \               /
               \             /
                v           v
          SpectralResidualFusion
                    |
                    v
             LayerNorm，512 维融合特征
                    |
          +---------+----------+
          |                    |
          v                    v
FC 训练监督：               导出融合特征
512 -> 256 -> 类别数              |
                                 v
                    StandardScaler + Linear SVM
```

## 3. 数据划分与评估协议

入口配置位于 `mffn_config.py`，主要实现位于 `pipeline.py`。

### 3.1 `strict` 协议（当前唯一协议）

当前 `EVALUATION_PROTOCOL = "strict"`，`paper` 协议已被移除。执行顺序为：

1. 先对原始重采样数据按 7:2:1 分层划分；
2. 分别对 train、val、test 执行预处理；
3. 仅对已经预处理的训练集执行 `KMeansSMOTE`；
4. 验证集和测试集保持原始类别分布。

**为什么移除 `paper` 协议？**

实验验证表明，`paper` 协议（先过采样再划分）存在严重的数据泄露问题：

| 数据集 | `paper` 协议 | `strict` 协议 | 差距 |
|--------|-------------|--------------|------|
| 品种 | 98.0% | <80% | >18% |
| 产地 | 96.12% | <80% | >16% |

`paper` 协议中，合成样本可能进入验证集和测试集，导致评估结果虚高。`strict` 协议确保验证集和测试集全部由原始样本组成，评估结果更真实可靠。

### 3.2 自定义 KMeans-SMOTE

`utils/oversampling.py` 中的 `KMeansSMOTE` 不是 `imbalanced-learn` 的现成实现。对每个非多数类，它会：

1. 在该类别内部执行 KMeans；
2. 仅使用样本数不少于 2 的簇；
3. 按可用簇样本数比例分配待合成数量；
4. 在同簇最近邻之间进行线性插值。

当配置簇数导致全部簇都是单样本簇时，代码会递减簇数重试。当前默认品种数据中，类别 7 从 8 个簇回退到 4 个簇；其余类别保持配置的 8 个簇。

`OVERSAMPLING_STRATEGY` 还支持 `median`，但当前默认是 `max`。当使用 `median` 时，代码不会下采样较大类别，因此不能将其描述为"所有类别都严格变为中位数"。

## 4. 光谱预处理

当前默认 `SPECTRAL_PREPROCESSING_METHOD = "snv+d1+sg"`，顺序由字符串从左到右决定：

1. SNV：逐样本减去自身均值，再除以自身标准差；
2. D1：使用 `np.diff(..., axis=1)` 计算相邻波段一阶差分，778 变为 777；
3. SG：使用 Savitzky-Golay 滤波，默认窗口 11、多项式阶数 2，长度不变。

主光谱分支和小波分支都以这份预处理后的光谱为输入；小波分解不直接作用于原始反射率。

`mms`、`ss`、`msc` 是依赖集合统计量的可选预处理。代码只在训练集拟合它们的缩放器或参考光谱，再应用到验证集和测试集。SNV、D1、SG 等逐样本操作则在每个集合内直接执行。

## 5. 小波辅助视图

实现位于 `utils/wavelet_features.py`。默认配置为：

```python
WAVELET_NAME = "db4"
WAVELET_LEVEL = 3
WAVELET_INCLUDE_DENOISED = False
```

对每条预处理光谱进行三级 DWT 后，`pywt.wavedec` 返回：

```text
[cA3, cD3, cD2, cD1]
```

代码通过"保留一个频带、其余系数清零、逆变换"的方式，重构为与输入等长的信号：

- 通道 1：`A3` 低频近似重构；
- 通道 2：`D3 + D2` 中低频细节重构；
- `D1` 最高频细节默认丢弃，以减轻高频噪声影响。

因此默认小波张量为 `[B, 2, 777]`。若 `WAVELET_INCLUDE_DENOISED = True`，还会追加 `A3 + D3 + D2` 的重构通道，形状变为 `[B, 3, 777]`。这里的"denoised"表示省略最高频 `D1` 后的重构，不是额外调用阈值去噪函数。

如果有效小波层数为 1，系数为 `[cA1, cD1]`，第二通道会包含 `D1`，因为没有更低层细节可以合并。

## 6. 编码器结构

### 6.1 主光谱编码器

默认主分支是 `models/spectral_extractors.py` 中的 `MultiScaleSpectralCNN`：

1. 三个并行 `Conv1d + BatchNorm1d + GELU` 分支，卷积核为 3、7、15，每支 48 通道；
2. 拼接为 144 通道，经 `1x1 Conv + BatchNorm + GELU` 融合为 192 通道；
3. 三个残差块。每个块为两层膨胀卷积、BatchNorm、GELU、光谱注意力、Dropout，并与输入残差相加后再 GELU；
4. 残差块 1 和 2 后各有一次 `AvgPool1d(2)`，残差块 3 后不池化；
5. 对末端特征同时做 8 区间自适应平均池化和最大池化；
6. 展平后投影到 512 维输出。

默认输入长度为 777 时：

```text
[B,1,777]
-> 三路 [B,48,777]
-> 拼接 [B,144,777]
-> 融合 [B,192,777]
-> 残差块1 -> AvgPool [B,192,388]
-> 残差块2 -> AvgPool [B,192,194]
-> 残差块3            [B,192,194]
-> AvgPool(8) 与 MaxPool(8)，沿通道拼接 [B,384,8]
-> 展平 [B,3072]
-> Linear(3072,512) -> LayerNorm -> GELU -> Dropout(0.25)
-> Linear(512,512)
```

三个残差块的参数分别是：

- 块 1：`kernel_size=7`，`dilation=1`，`dropout=0.08`；
- 块 2：`kernel_size=5`，`dilation=2`，`dropout=0.10`；
- 块 3：`kernel_size=3`，`dilation=4`，`dropout=0.12`。

### 6.2 小波编码器

默认小波分支使用 `AttentionCNN1D`，输入通道数由小波视图决定，当前为 2：

```text
Conv1d(2,128,k=7) -> BatchNorm -> GELU -> AvgPool -> SpectralAttention -> Dropout(0.10)
Conv1d(128,256,k=5) -> BatchNorm -> GELU -> AvgPool -> SpectralAttention -> Dropout(0.15)
Conv1d(256,128,k=3) -> BatchNorm -> GELU -> AvgPool -> SpectralAttention -> Dropout(0.20)
AdaptiveAvgPool1d(1) 与 AdaptiveMaxPool1d(1) -> 拼接 256 维
Linear(256,512) -> LayerNorm -> GELU -> Dropout(0.25) -> Linear(512,512)
```

默认输入长度 777 经过三次平均池化后的长度依次为 388、194、97。`SpectralAttention` 同时按通道和波长位置生成 Sigmoid 权重，并将二者相乘作用于特征图。

`IMAGE_DROPOUT` 只影响真实图像分支；小波编码器内部的 Dropout 固定为上述 0.10、0.15、0.20 和 0.25。

### 6.3 特征归一化

`MFFNINIRS._encode_modalities()` 会对两个编码器各自的 512 维输出执行 L2 归一化，然后才送入融合模块及辅助分类头。

## 7. 特征融合

默认 `FUSION_METHOD = "spectral_residual"`，实现为 `SpectralResidualFusion`。记小波特征为 `secondary`，主光谱特征为 `spectral`：

```text
secondary_delta = SecondaryAdapter(secondary)
gate_input = [secondary, spectral, |secondary - spectral|]
gate = sigmoid(GateMLP(gate_input))
gated_delta = gate * secondary_delta
secondary_scale = sigmoid(learnable_logit)
fused = spectral + secondary_scale * gated_delta
```

当前 `FEATURE_DIM=512`、`FUSION_HIDDEN_DIM=256`：

- 小波适配器：`512 -> 256 -> 512`，含 LayerNorm、GELU、`FUSION_DROPOUT`；
- 门控网络输入为 1536 维，输出为逐特征的 512 维 Sigmoid 门；
- 全局残差强度初始值为 `FUSION_INITIAL_SECONDARY_WEIGHT=0.05`，实际参数保存为其 logit；
- 融合结果随后经过 `fusion_norm = LayerNorm(512)`。

代码还保留 `concat`、`dwgff`、`acgf` 融合方法，当前默认不会实例化它们。

## 8. 训练目标、选模与最终分类器

### 8.1 FC 头与损失

融合特征先进入：

```text
feature_head:
512 -> Linear(256) -> LayerNorm -> GELU -> Dropout(0.15)

classifier:
Linear(256, num_classes)
```

`Dropout(0.15)` 来自 `max(secondary_dropout, FUSION_DROPOUT, 0.15)`；在默认小波模式下，`secondary_dropout=0`、`FUSION_DROPOUT=0.15`。

训练总损失为：

```text
总损失 = 标签平滑交叉熵
       + WAVELET_AUX_LOSS_WEIGHT * CE(小波辅助分类头)
       + SPECTRAL_AUX_LOSS_WEIGHT * CE(主光谱辅助分类头)
       + 可选 CenterLoss 项
```

默认值为：

- `LABEL_SMOOTHING = 0.05`；
- `USE_BRANCH_AUX_LOSS = True`；
- `WAVELET_AUX_LOSS_WEIGHT = 0.05`；
- `SPECTRAL_AUX_LOSS_WEIGHT = 0.00`；
- `USE_CENTER_LOSS = False`。

小波辅助分类头直接作用于 L2 归一化后的小波特征，因此其 0.05 加权交叉熵确实会向小波编码器传播梯度。权重较小只表示其对总目标的影响受限，不能表述为"辅助分支没有梯度"。

配置中存在 `USE_MODAL_ALIGN_LOSS` 与 `MODAL_ALIGN_LOSS_WEIGHT`，但当前所有 Python 代码都没有读取这两个配置项；模态对齐损失尚未实现。

### 8.2 优化、调度与早停

默认优化器为 AdamW，按模块使用三个参数组：

- 小波分支：`WAVELET_BACKBONE_LR = 1e-4`；
- 主光谱分支：`SPECTRAL_BACKBONE_LR = 1e-4`；
- 融合模块、LayerNorm、FC 头和辅助分类头：`FUSION_HEAD_LR = 1e-4`。

默认使用 `ReduceLROnPlateau`，`factor=0.5`、`patience=8`、`min_lr=1e-6`。当 `BEST_MODEL_METRIC="val_loss"` 时，它监控验证总损失；当取值为 `val_acc` 或当前默认的 `val_acc_or_loss` 时，它监控 `1 - val_accuracy`。

默认 `BEST_MODEL_METRIC="val_acc_or_loss"` 的最佳权重判定为：

- 验证准确率相对当前基准提高超过 `1e-4`；或
- 验证总损失降低超过 `1e-4`，且当前验证准确率不比当前基准低超过 `0.01`。

一旦保存最佳权重，当前 epoch 的验证 Accuracy 和验证 Loss 会作为下一轮联合判断的同一组基准，不会混用不同 epoch 的 Accuracy 与 Loss。至少训练 12 轮后，连续 14 轮未满足上述改进条件才早停。

### 8.3 Linear SVM

默认 `FINAL_CLASSIFIER="linear_svm"`、`FINAL_CLASSIFIER_FEATURE_MODE="fused"`、`FINAL_CLASSIFIER_TRAIN_SPLIT="trainval"`：

1. 恢复最佳神经网络权重；
2. 导出 train、val、test 的 512 维融合特征；
3. 合并 train 和 val 特征；
4. 对候选 `C=[0.03,0.05,0.1,0.2,0.5,1.0,2.0]` 分别执行分层交叉验证；
5. 每个候选都在 `Pipeline(StandardScaler, LinearSVC)` 内交叉验证，缩放器只在各训练折拟合；
6. 以默认 `f1_macro` 选择 `C`，在完整 train+val 特征上重拟合，再评估 test。

当前 `fused` 模式输入 SVM 的是 512 维融合特征。可选的 `all`、`hybrid`、`enhanced` 会拼接分支特征及交互特征，但默认不启用。

## 10. 文件调用关系

- `models/fusion.py`：当前实例化 `SpectralResidualFusion`；
- `utils/wavelet_features.py`：生成小波辅助视图；
- `utils/oversampling.py`：自定义 KMeans-SMOTE；
- `utils/image_encoding.py`：当前不生成编码图，但 `load_raw_dataset()` 通过其中的 `resample_series()` 将默认 1555 个原始波段重采样到 778；
- `models/image_backbones.py`：当前不会构建图像网络，但被模型模块和配置校验导入；
- `models/losses.py`：始终导入，默认 Center Loss 关闭时不实例化；
- `utils/pso_svm.py`：始终导入，默认 Linear SVM 路径不调用。

### 10.2 仅在其他配置或基准模式使用

- `utils/image_encoding.py` 中的 RP、GASF、GADF、MTF：`multimodal`、`image_only` 或相关基准实验；
- `models/image_backbones.py`：真实图像分支；
- `models/spectral_extractors.py` 中的 `PaperCNN1D`、LSTM、BiLSTM：骨干对比；
- `models/fusion.py` 中的 `concat`、`dwgff`、`acgf`：融合对比；
- `utils/pso_svm.py`：`FINAL_CLASSIFIER="pso_svm"`；
- `models/losses.py`：启用 Center Loss 时；
- `pipeline.py` 中的预处理、特征模式和论文复现实验套件：对应 `RUN_*` 开关开启时。

### 10.3 不在主训练调用链中的文件

- `修改过程.txt`：实验与修改记录；
- `项目模型架构说明.md`：本文档；
- `models/__init__.py`、`utils/__init__.py`：当前为空包初始化文件。

`encodings/` 是二维编码图像缓存目录；默认小波模式跳过编码图读取与生成。`results/` 是输出目录，当前工作区没有可由默认预训练加载逻辑直接使用的历史模型 `.pth` 文件；一次新的训练会在其中写入模型、曲线、混淆矩阵、ROC 和 CSV。

## 13. 数据集协议选择与大数据模式

### 13.1 协议选择策略

**重要更新**：`paper` 协议已被移除，所有数据集统一使用 `strict` 协议。实验验证表明 `paper` 协议存在严重的数据泄露问题（品种 98% → <80%，产地 96% → <80%）。

所有数据集的 `protocol` 已统一为 `"strict"`：

| 数据集 | 样本数 | 类别数 | 过采样 | 协议 | 大数据模式 |
|--------|--------|--------|--------|------|-----------|
| 品种 | 287 | 8 | `max` | `strict` | 关闭 |
| 产地 | 535 | 7 | `max` | `strict` | 关闭 |
| Apple | ~1000+ | 均衡 | 关闭 | `strict` | 开启 |
| Coffee | ~1000+ | 均衡 | 关闭 | `strict` | 开启 |
| Mango | 未知 | 未知 | `median` | `strict` | 关闭 |
| Diesel | 394 | 6 | `max` | `strict` | 关闭 |

### 13.2 大数据模式（`large_data_mode`）

对于样本量大且均衡的数据集（如 Apple、Coffee），建议启用 `large_data_mode=True`。启用后自动调整以下配置：

| 配置项 | 普通模式 | 大数据模式 |
|--------|---------|-----------|
| `APPLY_OVERSAMPLING` | 按数据集配置 | `False`（跳过过采样） |
| `EVALUATION_PROTOCOL` | `strict` | `strict`（不变） |
| `FEATURE_DIM` | 512 | 1024（更大容量） |
| `BATCH_SIZE` | 64 | 128（更大批量） |
| `EPOCHS` | 60 | 100（更多训练轮次） |

大数据模式的核心思想：
1. **跳过过采样**：大数据集样本足够，过采样不仅不必要，还会引入合成样本的偏差；
2. **使用 `strict` 协议**：先划分再预处理，测试集保持原始分布，评估更真实；
3. **增大模型容量**：特征维度从 512 提升到 1024，让模型有更多参数学习复杂模式；
4. **增大批量**：从 64 到 128，利用大数据集的优势稳定梯度；
5. **更多训练轮次**：从 60 到 100，配合早停机制，让模型充分收敛。

### 13.3 如何切换数据集

只需修改 `mffn_config.py` 中的 `DATASET_NAME` 一行：

```python
DATASET_NAME = "Apple"  # 可选: 品种 / 产地 / Apple / Coffee / Mango / Diesel
```

所有其他配置（协议、过采样策略、特征维度、批量大小等）都会自动从 `DATASET_CONFIGS` 注册表中读取。

### 13.4 协议一致性总结

| 方面 | 论文 | 旧 `paper` 协议（已移除） | 当前 `strict` 协议 |
|------|------|--------------------------|-------------------|
| 过采样时机 | 先过采样再划分 | 先过采样再划分 | 先划分，仅对训练集过采样 |
| 过采样策略 | 扩充到多数类大小 | `max` 策略 | 支持 `max`/`median` |
| 7:2:1 划分 | ✅ | ✅ | ✅ |
| 数据泄露风险 | 有（合成样本跨集） | 有（与论文一致） | ✅ 无（仅训练集过采样） |
| 评估真实性 | 一般（虚高） | 一般（虚高） | ✅ 更真实 |

**核心原则**：所有数据集统一使用 `strict` 协议，确保评估结果的真实性和可靠性。
